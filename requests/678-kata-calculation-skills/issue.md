# New Module 678: Kata — Building Calculation Skills

## Module Overview

- **Number:** 678
- **Folder:** `modules/678-kata-calculation-skills/`
- **Placement:** After module 650 — WinAPI MCP Toolbox
- **Skill statement:** Practice the full skill-creation loop — describe a calculation, let the agent generate a deterministic Python CLI script, write a SKILL.md, and verify the output — repeating with a new calculation theme each round.

## Problem This Module Solves

Engineers who learn about AI skills in module 090 often stop after one attempt. They never build the muscle memory to rapidly create new skills. This module runs a focused kata: a short, repeating loop where the user spins up multiple calculation skills in one session, guided by the agent. Because every script is deterministic (Python `math`, `statistics`, or stdlib only — no LLM calls inside the script), there is no ambiguity about correctness. Users learn to distinguish "what the model does" from "what the tool does", and they leave the module with 2-3 working skills they created themselves.

## Topics to Cover

- The kata concept applied to AI-assisted development: short loops, high repetition, clear feedback
- The skill-creation loop: describe → generate script → write SKILL.md → run → verify → commit
- Deterministic vs. generative computation — when to use each
- Python stdlib for calculations: `math`, `statistics`, `decimal`, `fractions`, `itertools`
- Script interface conventions: `argparse`, `--output`, `--format md|json|csv`
- SKILL.md schema: name, description, version, usage, example commands, notes
- Dependency bundling: how a skill packages its script alongside its instruction file
- Selecting the next kata theme: variation and progressive challenge

## Learning Outcome

Complete 2-3 full skill-creation cycles in one session: each cycle produces a working Python CLI script and a SKILL.md packaged together as a reusable AI skill. Each skill computes something meaningful (e.g., Fibonacci sequence, unit conversion table, statistics summary, amortization schedule) using only Python stdlib. After the module, the user can start any new calculation skill from scratch in under 10 minutes.

## Hands-on Walkthrough Outline

1. **Warm-up review** — Open the reference skill `instructions/calculate-trig-table/SKILL.md` and trace how the script, instruction file, and test fit together. Answer: what makes this skill "deterministic"?
2. **Round 1 — Theme assignment** — Agent assigns the first theme: *Fibonacci sequence table*. User describes the desired CLI interface to the agent (arguments: `--count`, `--output`, `--format`).
3. **Round 1 — Script generation** — Agent generates `scripts/calculate.py`. User runs it, inspects output, fixes any issues together with the agent.
4. **Round 1 — SKILL.md creation** — Agent drafts the SKILL.md for this skill. User reviews and edits the description, example commands, and notes sections.
5. **Round 1 — Verification** — User runs the script from the SKILL.md example commands verbatim to confirm they work exactly as documented.
6. **Round 2 — New theme** — Agent introduces a second theme (user picks from: unit conversion table, statistics summary, amortization schedule, prime sieve). Full loop repeated.
7. **Round 3 (stretch) — User-chosen theme** — User names any calculation they want. Agent and user build the skill together with minimal scaffolding.
8. **Debrief** — Review the three created skills side by side. Identify what's the same (SKILL.md schema, argparse, output formats) and what varies (domain logic). Discuss when to reach for a model vs. a script.
9. **Optional commit** — Stage all three skill folders and commit with a conventional commit message generated by the agent.

## Success Criteria

- ✅ At least two skills created: each has a working `scripts/calculate.py` and a complete `SKILL.md`
- ✅ Example commands in each SKILL.md run without modification and produce correct output
- ✅ User can explain the difference between deterministic and generative computation
- ✅ User can start the next skill creation loop independently without re-reading instructions
- ✅ (Stretch) Three skills committed to Git with agent-generated commit messages
