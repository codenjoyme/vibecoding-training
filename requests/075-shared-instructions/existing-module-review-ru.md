## Что уже есть

### Основная концепция (сильная — оставляем)

Модуль правильно формулирует ключевую мысль: **инструкции — это организационное знание, а не личная конфигурация**. Аналогия — когда Боб уходит из компании, его знания о промптах уходят вместе с ним — убедительна и понятна любой аудитории.

### Структура (по частям)

| Часть | Тема | Оценка |
|---|---|---|
| **Часть 1** | Проблема непоследовательности | ✅ Сильная. Хорошая таблица, показывающая, как Алиса/Боб/Кэрол получают разное качество AI. Чёткое бизнес-обоснование. |
| **Часть 2** | Структура папок общего репозитория | ✅ Добротная. Практичный шаблон `team-instructions/` с `instructions/`, `conventions/`, `templates/`. Включены команды оболочки. |
| **Часть 3** | Наполнение первой инструкции | ✅ Хорошая точка входа. Строится на результатах модуля 070, лёгкий первый шаг. |
| **Часть 4** | Дистрибуция через Git | ✅ Покрывает init, пуш на GitHub, fork/branch PR-воркфлоу. Практично и корректно. |
| **Часть 5** | Разрешение конфликтов | ✅ Полезно. Система из 4 правил приоритета (конкретное > общее, последний смерджённый побеждает, явное > неявное, при сомнениях открыть issue) — применимо на практике. |
| **Часть 6** | Стратегия внедрения | ✅ Паттерн COIN (Copy → Observe → Improve → Normalise) — самый уникальный и ценный элемент модуля. Очень практично для менеджеров. |

### Ожидаемый результат обучения

> "У вас есть шаблонная структура репозитория общих инструкций, вы понимаете воркфлоу instructions-as-code и у вас есть конкретный план внедрения, чтобы познакомить команду с общими инструкциями."

Это **хороший, достижимый результат** за 15 минут. Скоуп соответствующий.

## Что можно улучшить

### Пробелы по сравнению с исследованием

| Пробел | В исследовании | В текущем модуле |
|---|---|---|
| **Манифест / выбор навыков** | Папка `.manifest/` на уровне проекта с файлами `global.json`, `agents.json` и `<имя-проекта>.json` | Не упоминается |
| **Sparse checkout** | Частичное рабочее пространство, только нужные навыки | Не упоминается |
| **Анатомия навыка** | `SKILL.md` + `evals.json` *(будет позже)* + `README.md` на навык | Только `.agent.md`-файлы |
| **Модель владения** | Поле owner в README, ворота одобрения | Упомянуто вскользь в разрешении конфликтов |
| **IDE-агностичное ядро** | Центральный `skill.md`, тонкие IDE-обёртки | Частично осведомлён о VSCode/Cursor, нет чёткого паттерна |
| **Абстракция CLI** | Go-бинарник скрывает сложность Git | Ручные Git-команды на протяжении всего воркфлоу |
| **Валидация** | `evals.json` для тестирования качества инструкций | Не упоминается |

### Мелкие замечания

- Структура папок использует нейминг `.agent.md`, специфичный для VSCode; `skill.md` из исследования более IDE-агностичен
- Шаг `instructions pull` для потребителей кажется слабоватым — просто `git pull`, без упоминания обновлений sparse checkout
- Нет упоминания того, что происходит, когда центральный репозиторий вырастает до 50+ навыков — проблема обнаруживаемости
