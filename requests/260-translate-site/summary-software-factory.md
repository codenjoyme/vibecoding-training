# Саммари: How StrongDM's AI team build serious software without even looking at the code

**Автор:** Simon Willison | **Дата:** 7 февраля 2026 | **Источник:** [simonwillison.net](https://simonwillison.net/2026/Feb/7/software-factory/)

## Главная идея

Саймон Уиллисон описывает работу ИИ-команды StrongDM (3 человека), которая реализовала «Тёмную фабрику» — полностью автономный пайплайн разработки, где **ни один человек не пишет и не ревьюирует код**.

## Ключевые концепции StrongDM

### Мантра команды
- «Почему я делаю это?» — если задачу может сделать модель, человек не должен
- **Код не должен быть написан людьми**
- **Код не должен быть проревьюирован людьми**
- Бюджет: ~\/день на токены на инженера

### Сценарии как holdout-наборы
- Вдохновлено сценарным тестированием (Cem Kaner, 2003)
- Сценарии — сквозные пользовательские истории, хранятся **вне кодовой базы** (как holdout-набор в ML)
- LLM валидирует результат вероятностно: термин **«удовлетворённость»** — доля траекторий, удовлетворяющих пользователя
- Ключевой принцип: агент, пишущий код, не видит тесты

### Вселенная цифровых двойников (DTU)
- Поведенческие клоны сторонних сервисов: Okta, Jira, Slack, Google Docs/Drive/Sheets
- Создаются агентами по публичной API-документации как Go-бинарники
- Позволяют тестировать без rate-лимитов, квот, затрат на API
- Тысячи сценариев в час, включая опасные/невозможные режимы отказа
- **Создание такого клона всегда было возможно, но никогда не было экономически целесообразно** — агенты это изменили

### Дополнительные техники
- **Генная трансфузия** — извлечение паттернов из одной системы для использования в другой
- **Семпорты** — прямой перенос кода между языками
- **Пирамидные саммари** — многоуровневые резюме для агентов (короткие для обзора, детальные для погружения)

## Выпущенное ПО

- **[Attractor](https://github.com/strongdm/attractor)** — неинтерактивный кодирующий агент. Репозиторий не содержит кода — только 3 markdown-файла со спецификацией
- **[cxdb](https://github.com/strongdm/cxdb)** — AI Context Store (16K Rust + 9.5K Go + 6.7K TypeScript), хранилище историй в неизменяемом DAG

## Позиция Уиллисона

- Впечатлён демкой (посещал команду в октябре 2025, через 3 месяца после основания — уже всё работало)
- **Скептичен к стоимости** \/день — если это реально +\/месяц, это бизнес-модельный вопрос
- Лично использует Claude Max за \/месяц
- Считает главным вопросом: **как агенты могут доказать, что код работает, без построчного ревью**
- Отмечает точку перелома ноября 2025 (Claude Opus 4.5, GPT 5.2)
- Конкуренция меняется: фичи можно клонировать за часы работы агентов
