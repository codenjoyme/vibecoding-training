## Architecture (As Described)

- **Language:** Python (only firm tech requirement from expert; everything else delegated to the model).
- **Runtime:** Docker container; survives restarts; static files / SQLite-like DB stored on a mounted volume.
- **LLM backend:** Pluggable; currently uses GPT-4o via the GitHub Models / Copilot account (no premium-request burn). Model is configurable in `.env`.
- **Auth:** SSO via the corporate Azure AD account. On first launch a standard auth window opens (similar to corporate VPN auth). After that the app acts on behalf of the user.
- **Coverage:** 1:1 chats, group chats, meetings. Channels (Teams team channels) are NOT included — they require a separate Graph API and were left out of scope.
- **Notifications:** On startup the app auto-creates a dedicated chat/channel in the user's Teams and posts short notifications there (verbosity configurable).

## How the Summarization Logic Works

- On startup it scans the last N chats (default ~50, configurable).
- For each chat it pulls the last ~40 messages as a "bulk", produces a first summary with topic extraction.
- Then runs on a polling interval (configurable in minutes), looking for new messages in the chats it has access to.
- For each new message it does an incremental summary: previous summary + new message → updated summary (accumulative effect).
- Every 10 iterations it forces a full re-summary to avoid drift / loss of original meaning.
- User can pick exactly which chats to track — everything is opt-in per chat.
