# AI Workflow Decision Guide — Daily Tasks with the Right Tools

**Duration:** 45-60 minutes

**Skill:** Know exactly which AI tools to use, when to use them, and how to combine them for any daily development task — eliminating "what should I do here?" uncertainty.

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- AI tool selection framework: when to use Chat vs. MCP vs. CLI vs. agent delegation
- Scenario-based practice across 10 real developer workflows
- Decision flowchart: task type → recommended tool(s)
- Combining multiple AI tools in a single end-to-end workflow
- Building your personal AI workflow playbook from real tasks

## AI Tool Decision Framework

| Situation | Recommended Tool(s) |
|---|---|
| Vague requirement, needs clarification | AI interview (chat) → spec → GitHub issue |
| Bug with known symptom | Chat → locate cause → fix → baby-step commit |
| New developer joining project | Agent reads codebase → generates onboarding docs |
| Legacy codebase, unknown structure | SpecKit → reverse-docs → safe insertion points |
| Batch file processing (50+ files) | Agent mode → bulk automation |
| New product idea, 2-hour window | SpecKit → full-stack PoC |
| Reviewing past AI work | Export Chat Session → search → knowledge base |
| Repetitive weekly manual task | Identify pattern → CLI skill → automate |
| UI bug hunting | Chrome DevTools MCP → emulate user → report |
| Sprint planning, backlog creation | AI interview → prioritized GitHub issues |

## Learning Outcome

You will independently identify the right AI tool(s) for any daily task and execute the complete workflow without hesitation. You'll leave with a practiced decision framework and a personal AI workflow playbook based on real tasks from your own work.
