# Rapid Prototyping with SpecKit

**Duration:** 20-30 minutes  
**Skill:** Build a spec-driven PoC from idea to working full-stack prototype using SpecKit methodology

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- Spec-driven development methodology (SpecKit)
- SpecKit slash commands and their purpose
- Structuring specs, plans, data models, and tasks as files
- Setting up a parallel workspace for spec-driven practice
- Bootstrapping a full-stack Node.js + Docker project with SpecKit
- Walking through all SpecKit phases: specify → clarify → plan → tasks → analyze → implement → checklist

## Learning Outcome

Ability to apply the SpecKit spec-driven workflow to build a full-stack Node.js application in Docker — from a plain English idea through structured specification, planning, and AI-assisted implementation.

## Prerequisites

### Required Modules

- [110 — Development Environment Setup](../110-development-environment-setup/about.md) — Node.js and Docker must be installed
- [070 — Custom Instructions](../070-custom-instructions/about.md)
- [040 — Agent Mode & AI Mechanics](../040-agent-mode-under-the-hood/about.md)

### Required Skills & Tools

- GitHub Copilot with Agent Mode enabled (VSCode or Cursor)
- Node.js ≥ 20 and Docker installed and working

## When to Use

- Starting any greenfield project with AI assistance
- When complex features need structured planning before coding
- When you want AI to generate implementation tasks automatically
- When a team needs shared, documented specs as the source of truth
