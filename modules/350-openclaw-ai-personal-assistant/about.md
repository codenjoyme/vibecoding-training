# OpenClaw — AI Personal Assistant Platform

**Duration:** 5-7 minutes  
**Skill:** Install OpenClaw, connect an AI model, and interact with a personal AI assistant through a local Gateway, dashboard, and chat channels.

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- What OpenClaw is and how it differs from IDE-based AI assistants
- Installing OpenClaw CLI and running the onboarding wizard
- Configuring a model provider (Anthropic, OpenAI, Google, etc.)
- Using the Control UI dashboard for chat
- Exploring built-in tools: exec, browser, web_search, file I/O
- Connecting a chat channel (Telegram) to talk to your AI from any device
- Understanding the skill and plugin ecosystem

## Learning Outcome

You will have a running OpenClaw Gateway with a configured AI model, be able to chat with your assistant through the dashboard and Telegram, and understand how to extend its capabilities with tools, skills, and plugins.

## Prerequisites

### Required Modules

- [100 — Model Context Protocol (MCP)](../100-mcp-model-context-protocol/about.md)
- [108 — Token & API Key Management](../108-token-api-key-management/about.md)

### Required Skills & Tools

- Node.js 22.14+ installed (Node 24 recommended)
- An API key from a model provider (Anthropic, OpenAI, or Google)
- Terminal / command line familiarity
- Telegram account (optional, for channel integration)
- Windows (native or WSL2), macOS, or Linux

## When to Use

- When you want an AI assistant accessible beyond your IDE — from your phone, browser, or any chat app
- When you need an AI agent that can run shell commands, browse the web, and manage files autonomously
- When you want to expose AI capabilities to non-technical team members through familiar messaging apps
- When prototyping AI-powered workflows that go beyond code editing
