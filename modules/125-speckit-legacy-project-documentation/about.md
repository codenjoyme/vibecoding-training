# SpecKit for Legacy Projects — Reverse Documentation & Onboarding

**Duration:** 15-20 minutes  
**Skill:** Use SpecKit to analyze an existing codebase, generate structured documentation, and manage the project with spec-driven practices going forward

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- Reverse-engineering specs from existing code with SpecKit
- Generating structured documentation for a legacy project
- Creating a constitution, specs, plans, and data models from running code
- Bridging a legacy codebase onto SpecKit rails for future development
- Adding new features to an already-documented legacy project using spec-driven workflow

## Learning Outcome

Ability to take any existing project — even one with zero documentation — and produce a full set of SpecKit artifacts (constitution, specs, plans, data models, task lists). After this module, the project can be maintained and extended using the same spec-driven workflow as a greenfield SpecKit project.

## Prerequisites

### Required Modules

- [120 — Rapid Prototyping with SpecKit](../120-rapid-poc-prototyping-with-speckit/about.md) — You must understand the full SpecKit workflow before reverse-applying it
- [110 — Development Environment Setup](../110-development-environment-setup/about.md) — Node.js and Docker must be installed
- [070 — Custom Instructions](../070-custom-instructions/about.md)

### Required Skills & Tools

- GitHub Copilot with Agent Mode enabled (VSCode or Cursor)
- Node.js ≥ 20 and Docker installed and working
- An existing project with source code (your own or the sample provided)
- `uv` package manager installed (covered in module 120)

## When to Use

- You inherited a project with little or no documentation
- You want to onboard new team members quickly by generating structured specs
- You need to plan new features on a legacy codebase but lack understanding of how it works
- You want to bring an older project under spec-driven discipline without rewriting it
- You are migrating a project to a new team and need a knowledge transfer package
