# Shared Instructions & Team Conventions

**Duration:** 15 minutes

**Skill:** Create a shared instruction repository versioned in Git so every team member benefits when one person finds a better prompt.

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- Why individual prompting creates inconsistent team AI quality
- Shared instructions folder structure: conventions, code style, review checklists
- Git-based distribution: clone, update, PR workflow for instructions
- Conflict resolution when two instructions contradict
- Adoption strategy for teams new to shared instructions

## Learning Outcome

You have a template shared-instructions repository structure, understand instructions-as-code workflow, and have a concrete adoption playbook to introduce shared instructions to your team.
