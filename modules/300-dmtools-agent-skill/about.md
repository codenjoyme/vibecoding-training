# Installing Third-Party Agent Skills (DMtools)

**Duration:** 5-7 minutes

**Skill:** Install a third-party agent skill into your AI IDE and immediately gain access to a rich knowledge base and automation toolkit without writing a single line of code.

**👉 [Start hands-on walkthrough](walkthrough.md)**

## Topics

- What agent skills are and how they extend your AI assistant
- Installing the DMtools skill via one-line command (macOS/Linux) or manually (Windows)
- Verifying the skill is available in your AI assistant
- Understanding what DMtools gives you: 152+ MCP tools, 23 ready-made jobs, JS agents
- The core DMtools architecture: `dmtools.env` + `config.json` + `dmtools run`
- First conversation with your AI using the installed skill

## Learning Outcome

You will be able to install any third-party agent skill into Cursor or VS Code. You'll understand the difference between a *skill* (knowledge for AI) and a *CLI tool* (executable), know how to verify a skill is active, and have a working mental model of DMtools architecture to support all follow-up modules.

## Prerequisites

- Completed Module 090: AI Skills - Context, Tools & Memory
- Cursor or VS Code with GitHub Copilot installed
- A project folder open in your IDE (any folder works)
- Git Bash (Windows) or Terminal (macOS/Linux)

## When to Use

- When a teammate shares a new agent skill with you
- When you find a useful skill on [agentskills.io](https://agentskills.io) or GitHub
- When onboarding to a project that has its own skill package
- Before starting any DMtools-based module (this module is a prerequisite for all 300-series modules)
